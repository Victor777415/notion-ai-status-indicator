# Notion AI 桌面伴侣（Desktop Companion）

常驻置顶的小窗，悬浮在**所有应用**之上，实时显示所有 Notion AI 对话的状态（转圈 / 完成 / 空闲），点击任意一条即可跳回浏览器对应标签页。它通过本地 WebSocket（`ws://127.0.0.1:8787`）与 Chrome 扩展通信，**数据不出本机**。

这就是 Codex 那种效果的最后一块：即使你切到非浏览器应用，也能持续看到“正在转圈”的实时进度（系统通知只在“完成”时弹一下）。

## 运行前提
- 已安装 Node.js（建议 18+）。
- 已在 Chrome 加载本仓库扩展（0.2.0 及以上），且扩展正常工作。

## 安装与启动
在终端进入仓库后：

```bash
cd desktop
npm install
npm start
```

首次 `npm install` 会下载 Electron，可能需要几分钟。启动后屏幕右上角会出现一个小窗：
- 顶部显示「● 已连接扩展」表示与浏览器打通；若为「○ 等待扩展…」，请确认 Chrome 已开且扩展已启用（必要时 chrome://extensions ↗ 重载扩展）。
- 在浏览器里让 Notion AI 回复，小窗即实时显示转圈 → 完成。
- 点击列表某条 → 浏览器自动切到对应窗口/标签。
- 拖动顶部标题栏可移动窗口；「–」最小化、「×」退出。

## 工作原理
```
Notion 页面(content) ─→ 扩展后台(SW) ─→ WebSocket(127.0.0.1:8787) ─→ 桌面伴侣(Electron)
                                     ←─────── 点击某条回传 focus 指令 ←───────
```
- 桌面伴侣是 WebSocket **服务端**（MV3 扩展只能做客户端）。
- 扩展启动即尝试连接；未开桌面程序时静默重试，不影响页内宠物/角标/通知。

## 常见问题
- **一直“等待扩展…”**：确认 Chrome 在运行、扩展已启用；重载扩展后再看。
- **端口被占用**：若 8787 被占，改端口需同时改扩展 `src/background/service-worker.js` 的 `DESKTOP_WS_URL` 与 `desktop/main.js` 的 `WS_PORT`。
